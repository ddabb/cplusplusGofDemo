# 代理模式 (Proxy)

## 定义

代理模式为其他对象提供一种代理以控制对这个对象的访问。

## 适用场景

- 远程代理：为一个对象在不同的地址空间提供局部代表
- 虚拟代理：根据需要创建开销很大的对象
- 保护代理：控制对原始对象的访问
- 智能引用：取代简单指针，在访问对象时执行一些附加操作

## 结构

```
Subject (抽象主题)
 request()    # 声明接口

RealSubject (真实主题)
 request()    # 实现接口

Proxy (代理)
 Subject* subject_
 request()    # 控制对 RealSubject 的访问
```

## 优点

- 可以控制对对象的访问
- 可以在访问对象前后添加额外操作
- 可以实现延迟加载
- 可以实现访问控制

## 缺点

- 增加了系统的复杂度
- 可能会降低性能

## 代码示例

```cpp
class Subject {
public:
    virtual ~Subject() = default;
    virtual void request() const = 0;
};

class RealSubject : public Subject {
public:
    void request() const override {
        std::cout << "RealSubject::request()" << std::endl;
    }
};

class Proxy : public Subject {
private:
    RealSubject* subject_;
public:
    Proxy() : subject_(nullptr) {}
    ~Proxy() { delete subject_; }
    void request() const override {
        if (!subject_) {
            subject_ = new RealSubject();
        }
        subject_->request();
    }
};
```

## 实际应用

- 网络请求代理
- 图片懒加载
- 权限控制
- 日志记录
- 缓存代理
