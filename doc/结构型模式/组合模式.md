# 组合模式 (Composite)

## 定义

组合模式将对象组合成树形结构以表示"部分-整体"的层次结构。组合模式使得用户对单个对象和组合对象的使用具有一致性。

## 适用场景

- 你想表示对象的部分-整体层次结构
- 你希望用户忽略组合对象与单个对象的不同，用户将统一地使用组合结构中的所有对象

## 结构

```
Component (抽象组件)
 operation()    # 声明接口

Leaf (叶子节点)
 operation()    # 实现接口

Composite (组合节点)
 children_     # 子组件集合
 add(child)    # 添加子组件
 remove(child) # 移除子组件
 getChild(i)   # 获取子组件
 operation()   # 递归调用子组件的 operation()
```

## 优点

- 简化客户端代码：客户端可以统一处理单个对象和组合对象
- 易于扩展：可以方便地添加新的组件类型
- 符合开闭原则

## 缺点

- 可能会使设计过于一般化，难以限制组合中的组件类型
- 某些操作可能在叶子节点上没有意义

## 代码示例

```cpp
class Component {
public:
    virtual ~Component() = default;
    virtual void operation() const = 0;
    virtual void add(Component*) {}
    virtual void remove(Component*) {}
};

class Leaf : public Component {
public:
    void operation() const override {
        std::cout << "Leaf::operation()" << std::endl;
    }
};

class Composite : public Component {
private:
    std::vector<Component*> children_;
public:
    void add(Component* child) override {
        children_.push_back(child);
    }
    void operation() const override {
        for (auto child : children_) {
            child->operation();
        }
    }
};
```

## 实际应用

- 文件系统（文件和文件夹）
- GUI控件（容器和子控件）
- 组织机构（公司、部门、员工）
- 图形绘制（组合图形和基本图形）
