# 外观模式 (Facade)

## 定义

外观模式为子系统中的一组接口提供一个统一的高层接口。外观模式定义了一个高层接口，这个接口使得这一子系统更加容易使用。

## 适用场景

- 当你要为一个复杂子系统提供一个简单接口时
- 客户程序与多个子系统之间存在很大的依赖性。引入外观模式将子系统与客户以及其他子系统解耦，可以提高子系统的独立性和可移植性
- 当你需要构建一个层次结构的子系统时，使用外观模式定义子系统中每层的入口点

## 结构

```
Facade (外观类)
 SubsystemA* subA_
 SubsystemB* subB_
 SubsystemC* subC_
 operation()    # 协调各个子系统

SubsystemA/B/C (子系统类)
 operationX()   # 各自的操作
```

## 优点

- 简化了客户端与子系统之间的交互
- 降低了系统的耦合度
- 提高了系统的可维护性
- 符合迪米特法则

## 缺点

- 外观类可能会变得过于庞大
- 可能违反开闭原则

## 代码示例

```cpp
class SubsystemA {
public:
    void operationA() const {
        std::cout << "SubsystemA::operationA()" << std::endl;
    }
};

class Facade {
private:
    SubsystemA* subA_;
    SubsystemB* subB_;
public:
    Facade() : subA_(new SubsystemA()), subB_(new SubsystemB()) {}
    ~Facade() { delete subA_; delete subB_; }
    void operation() const {
        subA_->operationA();
        subB_->operationB();
    }
};
```

## 实际应用

- 计算机启动过程（BIOS、操作系统、应用程序）
- 电商平台的订单处理（库存、支付、物流）
- 编译器（词法分析、语法分析、代码生成）
- 数据库连接池管理
