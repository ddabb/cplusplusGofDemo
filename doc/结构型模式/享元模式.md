# 享元模式 (Flyweight)

## 定义

享元模式运用共享技术有效地支持大量细粒度的对象。

## 适用场景

- 一个应用程序使用了大量的对象
- 完全由于使用大量的对象，造成很大的存储开销
- 对象的大多数状态都可变为外部状态
- 如果删除对象的外部状态，那么可以用相对较少的共享对象取代很多组对象
- 应用程序不依赖于对象标识

## 结构

```
Flyweight (抽象享元)
 operation(extrinsicState)    # 声明接口

ConcreteFlyweight (具体享元)
 operation(extrinsicState)    # 实现接口

FlyweightFactory (享元工厂)
 flyweights_    # 享元对象缓存
 getFlyweight(key)    # 获取或创建享元对象

UnsharedConcreteFlyweight (非共享享元)
 operation(extrinsicState)    # 不共享的对象
```

## 优点

- 减少内存占用
- 提高性能
- 简化对象结构

## 缺点

- 增加系统复杂度
- 需要维护共享对象的状态

## 代码示例

```cpp
class Flyweight {
public:
    virtual ~Flyweight() = default;
    virtual void operation(const std::string& extrinsicState) const = 0;
};

class FlyweightFactory {
private:
    std::unordered_map<std::string, Flyweight*> flyweights_;
public:
    Flyweight* getFlyweight(const std::string& key) {
        if (flyweights_.find(key) == flyweights_.end()) {
            flyweights_[key] = new ConcreteFlyweight(key);
        }
        return flyweights_[key];
    }
};
```

## 实际应用

- 文本编辑器中的字符对象
- 图形系统中的形状对象
- 数据库连接池
- 线程池
- 游戏中的粒子系统
