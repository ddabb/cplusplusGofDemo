# 装饰器模式 (Decorator)

## 定义

装饰器模式动态地给一个对象添加一些额外的职责。就增加功能来说，装饰器模式相比生成子类更为灵活。

## 适用场景

- 在不影响其他对象的情况下，以动态、透明的方式给单个对象添加职责
- 处理那些可以撤销的职责
- 当不能采用生成子类的方式进行扩充时

## 结构

```
Component (抽象组件)
 operation()    # 定义接口

ConcreteComponent (具体组件)
 operation()    # 实现接口

Decorator (抽象装饰器)
 Component* component_
 operation()    # 委托给 component->operation()

ConcreteDecoratorA/B (具体装饰器)
 operation()    # 在调用 component->operation() 前后添加额外行为
```

## 优点

- 灵活地扩展对象功能
- 符合开闭原则
- 可以组合多个装饰器
- 避免类爆炸

## 缺点

- 增加了系统的复杂度
- 可能产生很多小对象

## 代码示例

```cpp
class Component {
public:
    virtual ~Component() = default;
    virtual void operation() const = 0;
};

class ConcreteComponent : public Component {
public:
    void operation() const override {
        std::cout << "ConcreteComponent::operation()" << std::endl;
    }
};

class Decorator : public Component {
protected:
    Component* component_;
public:
    Decorator(Component* comp) : component_(comp) {}
    void operation() const override {
        component_->operation();
    }
};

class ConcreteDecorator : public Decorator {
public:
    ConcreteDecorator(Component* comp) : Decorator(comp) {}
    void operation() const override {
        std::cout << "Before" << std::endl;
        Decorator::operation();
        std::cout << "After" << std::endl;
    }
};
```

## 实际应用

- Java IO流（BufferedInputStream等）
- GUI控件装饰（边框、滚动条等）
- 日志记录装饰器
- 性能监控装饰器
