# 建造者模式 (Builder)

## 定义

建造者模式将复杂对象的构建与它的表示分离，使得同样的构建过程可以创建不同的表示。

## 适用场景

- 当创建复杂对象的算法应该独立于该对象的组成部分以及它们的装配方式时
- 当构造过程必须允许被构造的对象有不同的表示时

## 结构

```
Director (指挥者)
 construct(builder)    # 使用 Builder 创建产品

Builder (抽象建造者)
 buildPartA()
 buildPartB()
 buildPartC()
 getResult()           # 返回 Product

ConcreteBuilder (具体建造者)
 buildPartA()
 buildPartB()
 buildPartC()
 getResult()

Product (产品)
 partA
 partB
 partC
```

## 优点

- 将复杂对象的创建过程分解为多个简单步骤
- 允许创建不同表示的产品
- 解耦了产品的构建和表示
- 符合单一职责原则

## 缺点

- 产品必须有共同点，使用范围受到限制
- 如果产品内部变化复杂，会导致建造者类也很复杂

## 代码示例

```cpp
class Product {
public:
    void addPart(const std::string& part) {
        parts_.push_back(part);
    }
    
    void show() const {
        for (const auto& part : parts_) {
            std::cout << part << std::endl;
        }
    }
    
private:
    std::vector<std::string> parts_;
};

class Builder {
public:
    virtual ~Builder() = default;
    virtual void buildPartA() = 0;
    virtual void buildPartB() = 0;
    virtual Product* getResult() = 0;
};
```

## 实际应用

- 生成器模式常用于创建复杂对象，如：
  - 文档生成器
  - 报表生成器
  - 图形编辑器
  - 游戏角色创建
  - 数据库查询构建器
