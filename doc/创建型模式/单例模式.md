# 单例模式 (Singleton)

## 定义

单例模式确保一个类只有一个实例，并提供一个全局访问点来访问这个实例。

## 适用场景

- 当类只能有一个实例，且必须从一个访问点访问时
- 当这个唯一实例应该通过子类化扩展，且用户无需修改代码就能使用扩展的实例时

## 结构

```
Singleton
 私有构造函数
 私有静态成员变量 (唯一实例)
 公有静态方法 (获取实例)
```

## 实现方式

### 1. 饿汉式 (Eager Initialization)

在类加载时就创建实例，线程安全，但可能浪费资源。

### 2. 懒汉式 (Lazy Initialization)

在第一次使用时才创建实例，需要考虑线程安全问题。

### 3. 双重检查锁定 (Double-Checked Locking)

在懒汉式基础上，通过双重检查提高性能。

### 4. 静态局部变量 (Meyers Singleton)

C++11及以上版本保证静态局部变量初始化是线程安全的。

## 优点

- 确保只有一个实例，节省资源
- 提供全局访问点
- 避免重复创建对象

## 缺点

- 违反单一职责原则
- 可能导致测试困难
- 可能隐藏依赖关系

## 代码示例

```cpp
// C++11 线程安全的单例模式
class Singleton {
public:
    static Singleton& getInstance() {
        static Singleton instance;
        return instance;
    }
    
    Singleton(const Singleton&) = delete;
    Singleton& operator=(const Singleton&) = delete;
    
    void doSomething() {
        // 业务逻辑
    }
};
```

## 实际应用

- 数据库连接池
- 日志管理器
- 配置管理器
- 缓存管理器
