# 抽象工厂模式 (Abstract Factory)

## 定义

抽象工厂模式提供一个创建一系列相关或相互依赖对象的接口，而无需指定它们具体的类。

## 适用场景

- 当需要创建的对象是一系列相互关联或相互依赖的产品族时
- 当系统需要独立于它的产品创建、组合和表示时
- 当系统要由多个产品系列中的一个来配置时
- 当需要强调一系列相关产品对象的设计以便进行联合使用时
- 当提供一个产品类库，只想显示它们的接口而不是实现时

## 结构

```
AbstractFactory (抽象工厂)
 createProductA()    # 返回 AbstractProductA
 createProductB()    # 返回 AbstractProductB

ConcreteFactory1 (具体工厂1)
 createProductA()    # 返回 ProductA1
 createProductB()    # 返回 ProductB1

ConcreteFactory2 (具体工厂2)
 createProductA()    # 返回 ProductA2
 createProductB()    # 返回 ProductB2

AbstractProductA (抽象产品A)
 operationA()

AbstractProductB (抽象产品B)
 operationB()

ProductA1, ProductA2 (具体产品A)
ProductB1, ProductB2 (具体产品B)
```

## 优点

- 隔离了具体类的生成，客户端不需要知道什么被创建
- 保证了产品族内的一致性
- 增加新的产品族很方便

## 缺点

- 增加新的产品等级结构很困难，需要修改抽象工厂接口
- 类的数量较多

## 代码示例

```cpp
class AbstractProductA {
public:
    virtual ~AbstractProductA() = default;
    virtual void operation() const = 0;
};

class AbstractProductB {
public:
    virtual ~AbstractProductB() = default;
    virtual void operation() const = 0;
};

class AbstractFactory {
public:
    virtual ~AbstractFactory() = default;
    virtual AbstractProductA* createProductA() const = 0;
    virtual AbstractProductB* createProductB() const = 0;
};
```

## 实际应用

- GUI框架中的控件套件（按钮、文本框等）
- 数据库访问层（不同数据库的驱动）
- 跨平台应用（不同平台的实现）
- 游戏开发中的资源管理（不同类型的资源）
