# 中介者模式 (Mediator)

## 定义

中介者模式用一个中介对象来封装一系列的对象交互。中介者使各对象不需要显式地相互引用，从而使其耦合松散，而且可以独立地改变它们之间的交互。

## 适用场景

- 一组对象以定义良好但是复杂的方式进行通信
- 想定制一个分布在多个类中的行为，而又不想生成太多的子类
- 对象之间存在大量的交互，导致系统难以理解和维护

## 结构

```
Mediator (中介者接口)
 notify(Colleague, event)    # 处理同事对象的通知

ConcreteMediator (具体中介者)
 colleagues_    # 同事对象列表
 register(Colleague)
 notify(Colleague, event)    # 协调同事对象

Colleague (同事接口)
 Mediator* mediator_
 send(message)               # 发送消息

ConcreteColleague (具体同事)
 send(message)               # 通过中介者发送
 receive(message)            # 接收消息
```

## 优点

- 减少对象间的耦合
- 集中控制交互
- 简化对象协议
- 符合开闭原则

## 缺点

- 中介者可能变得过于复杂
- 可能成为性能瓶颈

## 代码示例

```cpp
class Colleague;

class Mediator {
public:
    virtual ~Mediator() = default;
    virtual void notify(Colleague* sender, const std::string& event) const = 0;
};

class Colleague {
protected:
    Mediator* mediator_;
public:
    explicit Colleague(Mediator* mediator) : mediator_(mediator) {}
    virtual void send(const std::string& message) = 0;
};
```

## 实际应用

- 聊天室系统
- 图形界面组件通信
- 微服务架构中的消息总线
- 飞机调度系统
- 智能家居控制中心
