# 观察者模式 (Observer)

## 定义

观察者模式定义了对象间的一种一对多依赖关系，当一个对象的状态发生改变时，所有依赖它的对象都得到通知并被自动更新。

## 适用场景

- 当一个抽象模型有两个方面，其中一个方面依赖于另一个方面
- 当对一个对象的改变需要同时改变其他对象，而不知道具体有多少对象有待改变时
- 当一个对象必须通知其他对象，而它又不能假定其他对象是谁

## 结构

```
Subject (主题)
 observers_    # 观察者列表
 attach(observer)    # 添加观察者
 detach(observer)    # 移除观察者
 notify()    # 通知所有观察者

Observer (观察者)
 update(subject)    # 更新方法

ConcreteSubject (具体主题)
 state_    # 状态
 getState()    # 获取状态
 setState(state)    # 设置状态并通知

ConcreteObserver (具体观察者)
 update(subject)    # 实现更新逻辑
```

## 优点

- 松耦合：主题和观察者之间是松耦合的
- 扩展性：可以方便地添加新的观察者
- 符合开闭原则

## 缺点

- 如果观察者数量过多，通知可能会耗时
- 如果观察者和主题之间存在循环依赖，可能导致系统崩溃

## 代码示例

```cpp
class Observer {
public:
    virtual ~Observer() = default;
    virtual void update() = 0;
};

class Subject {
private:
    std::vector<Observer*> observers_;
public:
    void attach(Observer* observer) {
        observers_.push_back(observer);
    }
    void notify() const {
        for (auto observer : observers_) {
            observer->update();
        }
    }
};
```

## 实际应用

- 新闻订阅系统
- 事件处理系统
- 数据绑定
- MVC框架
- 发布/订阅模式
