# 访问者模式 (Visitor)

## 定义

访问者模式表示一个作用于某对象结构中各元素的操作。它使你可以在不改变各元素类的前提下定义作用于这些元素的新操作。

## 适用场景

- 对象结构中对象对应的类很少改变，但经常需要在此结构上定义新的操作
- 需要对一个对象结构中的对象进行很多不同的并且不相关的操作，而你想避免让这些操作"污染"这些对象的类

## 结构

```
Visitor (访问者接口)
 visitElementA(element)    # 访问元素A
 visitElementB(element)    # 访问元素B

ConcreteVisitor (具体访问者)
 visitElementA(element)    # 具体实现
 visitElementB(element)    # 具体实现

Element (元素接口)
 accept(visitor)           # 接受访问者

ConcreteElementA/B (具体元素)
 accept(visitor)           # visitor->visitElementX(this)

ObjectStructure (对象结构)
 elements_
 addElement(element)
 accept(visitor)           # 遍历所有元素接受访问
```

## 优点

- 符合开闭原则，添加新操作只需添加新访问者
- 集中相关操作，分散无关操作
- 简化元素类
- 便于数据结构与操作分离

## 缺点

- 如果元素类经常变化，访问者接口需要频繁修改
- 破坏了元素的封装性
- 可能导致类爆炸

## 代码示例

```cpp
class Element;

class Visitor {
public:
    virtual ~Visitor() = default;
    virtual void visitElementA(const Element* element) const = 0;
    virtual void visitElementB(const Element* element) const = 0;
};

class Element {
public:
    virtual ~Element() = default;
    virtual void accept(const Visitor* visitor) const = 0;
};
```

## 实际应用

- 编译器语法树遍历
- XML/HTML文档解析
- 图形编辑器操作
- 报表生成
- 对象结构的不同视角展示
