# 策略模式 (Strategy)

## 定义

策略模式定义一系列算法，把它们封装起来，并使它们可以相互替换。策略模式使得算法可以独立于使用它的客户而变化。

## 适用场景

- 许多相关的类仅仅是行为有异
- 需要使用一个算法的不同变体
- 算法使用客户不应该知道的数据
- 一个类定义了多种行为，并且这些行为在这个类的操作中以多个条件语句的形式出现

## 结构

```
Context (上下文)
 Strategy* strategy_
 executeStrategy()    # 调用 strategy->algorithm()

Strategy (策略接口)
 algorithm()    # 声明算法接口

ConcreteStrategyA/B/C (具体策略)
 algorithm()    # 实现具体算法
```

## 优点

- 消除条件语句
- 符合开闭原则
- 提高代码复用性
- 易于扩展

## 缺点

- 增加了类的数量
- 客户端需要知道所有策略

## 代码示例

```cpp
class Strategy {
public:
    virtual ~Strategy() = default;
    virtual void algorithm() const = 0;
};

class Context {
private:
    Strategy* strategy_;
public:
    Context(Strategy* strategy) : strategy_(strategy) {}
    void execute() const {
        strategy_->algorithm();
    }
};
```

## 实际应用

- 排序算法策略
- 支付方式策略
- 日志记录策略
- 压缩算法策略
- 路由算法策略
