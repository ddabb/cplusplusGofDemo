# 责任链模式 (Chain of Responsibility)

## 定义

责任链模式使多个对象都有机会处理请求，从而避免请求的发送者和接收者之间的耦合关系。将这些对象连成一条链，并沿着这条链传递该请求，直到有一个对象处理它为止。

## 适用场景

- 有多个对象可以处理一个请求，哪个对象处理该请求运行时刻自动确定
- 你想在不明确指定接收者的情况下，向多个对象中的一个提交一个请求
- 可处理一个请求的对象集合应被动态指定

## 结构

```
Handler (抽象处理器)
 Handler* successor_    # 后继处理器
 setSuccessor(handler)  # 设置后继
 handleRequest(request) # 处理请求

ConcreteHandlerA/B/C (具体处理器)
 handleRequest(request) # 处理请求或传递给后继
```

## 优点

- 解耦：请求发送者和接收者解耦
- 灵活性：可以动态调整责任链
- 符合开闭原则

## 缺点

- 请求可能不被处理
- 可能导致性能问题
- 调试困难

## 代码示例

```cpp
class Handler {
protected:
    Handler* successor_;
public:
    Handler() : successor_(nullptr) {}
    void setSuccessor(Handler* successor) {
        successor_ = successor;
    }
    virtual void handleRequest(int request) const {
        if (successor_) {
            successor_->handleRequest(request);
        }
    }
};

class ConcreteHandler : public Handler {
public:
    void handleRequest(int request) const override {
        if (request == 1) {
            std::cout << "Handled by ConcreteHandler" << std::endl;
        } else {
            Handler::handleRequest(request);
        }
    }
};
```

## 实际应用

- 审批流程
- 日志记录
- 异常处理
- 事件处理
- 过滤器链
