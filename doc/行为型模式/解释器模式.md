# 解释器模式 (Interpreter)

## 定义

解释器模式给定一个语言，定义它的文法的一种表示，并定义一个解释器，这个解释器使用该表示来解释语言中的句子。

## 适用场景

- 当语言的文法较为简单，且执行效率不是关键问题时
- 当需要一个简单的语法解释器，且语法规则相对固定时
- 当需要将一种语言转换为另一种语言时

## 结构

```
AbstractExpression (抽象表达式)
 interpret(context)    # 解释方法

TerminalExpression (终结符表达式)
 interpret(context)    # 解释终结符

NonTerminalExpression (非终结符表达式)
 left_    # 左子表达式
 right_   # 右子表达式
 interpret(context)    # 解释非终结符

Context (上下文)
 data_    # 上下文数据
 存储解释器需要的信息
```

## 优点

- 可扩展性强，容易添加新的表达式
- 语法结构清晰
- 符合开闭原则
- 易于实现简单的语法

## 缺点

- 对于复杂的语法，解释器会变得非常复杂
- 执行效率较低
- 类的数量会急剧增加

## 代码示例

```cpp
class Context;

class AbstractExpression {
public:
    virtual ~AbstractExpression() = default;
    virtual int interpret(const Context& context) const = 0;
};

class TerminalExpression : public AbstractExpression {
private:
    std::string value_;
public:
    explicit TerminalExpression(const std::string& value) : value_(value) {}
    int interpret(const Context& context) const override {
        return std::stoi(value_);
    }
};
```

## 实际应用

- 编译器和解释器
- 正则表达式引擎
- SQL解析器
- 配置文件解析
- 脚本语言解释器
