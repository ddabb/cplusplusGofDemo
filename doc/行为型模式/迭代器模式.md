# 迭代器模式 (Iterator)

## 定义

迭代器模式提供一种方法顺序访问聚合对象中的各个元素，而又不暴露该对象的内部表示。

## 适用场景

- 访问一个聚合对象的内容而无需暴露它的内部表示
- 支持对聚合对象的多种遍历
- 为遍历不同的聚合结构提供一个统一的接口

## 结构

```
Iterator (迭代器接口)
 hasNext()    # 是否有下一个元素
 next()       # 返回下一个元素
 currentItem() # 返回当前元素

ConcreteIterator (具体迭代器)
 Aggregate* aggregate_
 currentIndex_
 实现迭代器接口

Aggregate (聚合接口)
 createIterator()    # 创建迭代器

ConcreteAggregate (具体聚合)
 items_    # 元素集合
 createIterator()    # 返回 ConcreteIterator
```

## 优点

- 简化了聚合类
- 支持多种遍历方式
- 符合单一职责原则

## 缺点

- 增加了类的数量
- 可能导致性能问题

## 代码示例

```cpp
template<typename T>
class Iterator {
public:
    virtual ~Iterator() = default;
    virtual bool hasNext() const = 0;
    virtual T next() = 0;
};

template<typename T>
class Aggregate {
public:
    virtual ~Aggregate() = default;
    virtual Iterator<T>* createIterator() const = 0;
};
```

## 实际应用

- STL容器的迭代器
- 数据库游标
- 文件遍历
- 图形遍历
- 树形结构遍历
