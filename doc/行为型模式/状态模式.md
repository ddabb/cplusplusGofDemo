# 状态模式 (State)

## 定义

状态模式允许一个对象在其内部状态改变时改变它的行为。对象看起来似乎修改了它的类。

## 适用场景

- 一个对象的行为取决于它的状态，并且它必须在运行时刻根据状态改变它的行为
- 一个操作中含有庞大的多分支条件语句，且这些分支依赖于该对象的状态

## 结构

```
Context (上下文)
 State* state_
 setState(state)    # 设置状态
 request()          # 委托给 state->handle()

State (状态接口)
 handle(context)    # 处理请求

ConcreteStateA/B/C (具体状态)
 handle(context)    # 处理请求并可能改变状态
```

## 优点

- 将状态相关的行为封装到状态类中
- 消除复杂的条件语句
- 符合开闭原则
- 提高代码可读性

## 缺点

- 增加了类的数量
- 可能导致状态转换逻辑分散

## 代码示例

```cpp
class State {
public:
    virtual ~State() = default;
    virtual void handle(class Context& context) const = 0;
};

class Context {
private:
    State* state_;
public:
    Context(State* state) : state_(state) {}
    void setState(State* state) {
        delete state_;
        state_ = state;
    }
    void request() const {
        state_->handle(*this);
    }
};
```

## 实际应用

- 电梯状态管理
- 订单状态流转
- 游戏角色状态
- TCP连接状态
- 工作流状态机
