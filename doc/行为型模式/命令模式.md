# 命令模式 (Command)

## 定义

命令模式将请求封装成对象，从而使得你可以用不同的请求对客户进行参数化，对请求排队或记录请求日志，以及支持可撤销的操作。

## 适用场景

- 需要将请求发送者和请求执行者解耦
- 需要在不同时间指定、排列和执行请求
- 需要支持撤销操作
- 需要支持日志记录和事务

## 结构

```
Command (命令接口)
 execute()    # 执行命令
 undo()       # 撤销命令

ConcreteCommand (具体命令)
 Receiver* receiver_
 execute()    # 调用 receiver->action()
 undo()       # 调用 receiver->undoAction()

Invoker (调用者)
 Command* command_
 setCommand(command)    # 设置命令
 executeCommand()       # 执行命令

Receiver (接收者)
 action()        # 执行操作
 undoAction()    # 撤销操作
```

## 优点

- 解耦：请求发送者和执行者解耦
- 可扩展性：容易添加新命令
- 支持撤销和重做
- 支持日志和事务

## 缺点

- 增加了类的数量
- 可能导致系统过于复杂

## 代码示例

```cpp
class Command {
public:
    virtual ~Command() = default;
    virtual void execute() const = 0;
    virtual void undo() const = 0;
};

class Receiver {
public:
    void action() const {
        std::cout << "Receiver::action()" << std::endl;
    }
};

class ConcreteCommand : public Command {
private:
    Receiver* receiver_;
public:
    ConcreteCommand(Receiver* receiver) : receiver_(receiver) {}
    void execute() const override {
        receiver_->action();
    }
};
```

## 实际应用

- 菜单操作
- 按钮点击
- 撤销/重做功能
- 事务管理
- 任务队列
