## 编写规则

### 代码注释规范

1. **类注释**: 每个类必须有块注释，说明类的用途、设计模式角色、核心职责
2. **成员函数注释**: 每个公有成员函数必须有块注释，说明函数功能、参数含义、返回值
3. **关键变量注释**: 关键成员变量和局部变量需要单行注释说明用途
4. **构造函数/析构函数注释**: 说明初始化和清理逻辑
5. **设计模式说明**: 在demo函数开头添加设计模式的核心思想说明
6. **代码逻辑注释**: 复杂逻辑分支和算法需要注释说明

### C++语法规范

1. 使用C++20标准特性
2. 使用智能指针 (`std::unique_ptr`, `std::shared_ptr`)
3. 使用 `override` 关键字明确重写
4. 使用 `= default`, `= delete` 明确特殊成员函数
5. 使用命名空间组织代码
6. 代码风格一致，缩进4空格

## 编译说明

### 环境要求
- C++20编译器（MSVC 2019+/GCC 10+/Clang 10+）
- CMake 3.16+
- SFML 2.5（用于GUI模式，可选）

### 编译步骤

#### 控制台版本（推荐，无需SFML）
```bash
mkdir build && cd build
cmake .. -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Release
cmake --build . --config Release
bin/DesignPatternDemo.exe
```

或者直接双击运行 `build_console.bat`

#### GUI版本（需要SFML）
```bash
mkdir build && cd build
cmake .. -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Release
cmake --build . --config Release
bin/DesignPatternDemo.exe
```

| 文件 | 说明 |
